최성일 Sass를 활용한 효율적인 CSS
1. Sass소개 및 설치
  1. Node.js설치 (npm이라는 패키지 관리자 툴을 사용할 수 있음)
    • Node.js 링크
    • 안전화 버전인 LTS를 다운로드를 한다.
    • 설치가 완료되면 Next만 누르면 된다.
    • 컴퓨터를 재부팅 후 cmd 터미널 창을 연 후 "node -v" 입력
    •                         node -v
    • 노드 버전이 뜨면 성공이다.
  2. 2단계 Dart 버전 Sass 설치(최신버전)
    • cmd 터미널 창을 연 후 "npm install sass -g" 입력
    •                         npm install sass -g
    • 여기서 -g는 글로벌 이라는 뜻이다. 시스템 전역에 설치하겠다.
    • 사스가 잘 설치됐는지 확인하려면 "sass --version"을 입력한다
    • 버전이 뜨면 잘 설치가 된것이다..
2. Sass 컴파일
  1. Sass
    • Sass는 css의 전처리기다.(작성은 sass로 읽는것은 sass에서 변환이 된 css로 읽는다)
    • 브라우저는 일반 css스타일만 읽는다
    • 폴더 sass를 만들고 그 안에 이름.sass 파일을 만든 후 다음과 같이 아래 코드를 입력해 본다. 
                          
                      * 
                        margin: 0px
                        padding: 0px
    • 그리고 터미널 창을 열어서 "sass --watch sass/style.sass css/style.css" 입력
    •                     
                      sass --watch sass/style.sass css/style.css
    • 위 구문은 "sass --감시하다 해당폴더를 그리고 css폴더에 style.css를 만든다
    • 엔터를 치고 명령어가 뜨면 잘 된것이다. 그러면 css 폴더와 그 안에 style.css 그리고 style.css.map 파일이 생긴다.
    • 터미널 창에서 "ctrl + c" 를 누르면 sass의 와칭이 풀린다.
  2. SCSS
    • Sass의 불편한 점을 개선해서 만든 것
    • 컴파일 하는 방법은 sass와 같다.
    • 터미널 창을 열어서 "sass --watch scss/style.scss css/style.css" 입력
    •                       
                        sass --watch scss/style.scss css/style.css
3. Sass map파일의 이해
  1. map 파일의 이해
    • map = 지도
    • scss에서 작업을 하면 실제 브라우저에서 읽고 있는 파일은 css 파일이다.
    • map이란 파일이 없으면 scss에서 어떤 클래스에 스타일을 주는데 그게 13번쨰 줄이라면.. css 에서는 10번째 줄에 있을 수 있다.
    • 개발자 도구(f12)로 창을 열여보면 그 클래스 스타일의 원본 위치가 어디에 있는지 알려주는 역할을 한다.

07. mixin
  1. mixin : 자주 만드는 UI 컴포넌트를 해당 기능을 빠르게 만들어 주는 css 꾸러미(패키지)를 만드는 기술
    • script의 함수와 비슷한 기능이라 할 수 있다
    • 자주 사용하는 기능을 명령어 하나로 빠르게
    • 공통적인 ui를 mixin 형태로 만들어 놓으면 편함
    • @mixin 변수이름(){}
    • _mixin.scss
    • _mixin.scss 
                          
                      a {
                      @include button();
                      }
    • 기본적인 방법은 쉽다. 하지만 기본 양식은 똑같은데 버튼의 가로길이만 바꾸고 싶다면 ?? 변수를 활용하면 된다.
    • @mixin 부분 괄호 안에는 변수를 만들고 바꾸고 싶은 css 값에도 변수를 넣는다.
    • @liclude 하는 변수쪽에는 값을 넣어주면 된다.
    • mixin scss부분 
                          
                      @mixin button($wid){
                      width: $wid;
                      height: 40px;
                      border-radius: 20px;
                      background-color: $bgButton;
                      font-size: 12px;
                      line-height: #fff;
                      display: flex;
                      justify-content: center;
                      align-items: center;
                      }
    • style scss부분 
                            
                        a {
                        @include button(150);
                        }
    • 그럼 각각 버튼 길이를 다르게 하고싶다면 ?

@each
  1. each 문
    • for문은 숫자밖에 안되지만 each문은 다된다.
    • #{} 은 괄호 안 변수를 넣을때 사용한다.
  2.                 
                    @each $el in $bgs{
                    $num: index($bgs, $el);
                    &:nth-of-type(#{$num}) {
                    transition-delay:$interval * $num;
                    a{
                    @include button($bg:$el);
                    }
                    }
                    }
